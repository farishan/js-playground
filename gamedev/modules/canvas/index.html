Tools - Canvas

Canvas is where we draw all of the game graphics.

How to define a canvas:

  • <canvas id="canvasId"></canvas> 
  •           
            const $canvas = document.createElement('canvas');
            $canvas.id = 'canvasId';

Notes:

  • Canvas constructor should only contain metadata of "the real canvas" seen by player.
    • The goal is to prevent the full canvas DOM to be passed between game runtime to save memory.
    • We can use something like "canvas manager" to save the id, and pass the id instead of the full object properties.
  • Your game entities is not "live" in a canvas. But, they have some "canvas-renderable"/graphical properties like dimension, color, or sprite.

While "the real canvas" managed by CanvasRenderer.

[draft] Canvas layer types, ordered by z-index:

  1. static layer: background
  2. dynamic layer: animated background entities
  3. dynamic layer: animated/controlled game entities
  4. dynamic layer: user interaction (mouse)
