The Goal

What is my dream game?

  • My dream game is a complex one, so a JavaScript module feature is a must.
  • My dream game is a simple browser game that need no webserver. Just click index.html file and play.
  • With 2 points above, I choose to use RequireJS as my module loader.

About Modules

  • MyModule.js means it is a pure javascript object. Doesn't mean it can be imported.
  • MyModule.module.js means it is a javascript module that can be imported. In my game development context, it should be imported/loaded with RequireJS.

If I need to use module in complex dev enviroment like React/Vue/Svelte, I can't just use pure javascript object. I need to convert the RequireJS module to matching module format, because I need to use module, not object. Module is module, wether it is in RequireJS format (AMD-based) or CommonJS or whatever module format.
